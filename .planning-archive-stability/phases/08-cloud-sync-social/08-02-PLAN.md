**Step 3: Create `src/sync/service.ts`** — Core sync logic (per D-06, D-07, D-08)

```typescript
import type { SQLiteDatabase } from 'expo-sqlite';
import type { SupabaseClient } from '@supabase/supabase-js';
import type { TableSyncConfig, SyncResult } from './types';
import { SYNC_TABLE_CONFIGS } from './tables';

let syncInProgress = false; // Sync lock (pitfall #5)

export async function syncAll(
  db: SQLiteDatabase,
  supabase: SupabaseClient,
  userId: string
): Promise<SyncResult> {
  if (syncInProgress) return { pushed: 0, pulled: 0, errors: ['Sync already in progress'] };
  syncInProgress = true;
  
  const totalResult: SyncResult = { pushed: 0, pulled: 0, errors: [] };
  
  try {
    for (const config of SYNC_TABLE_CONFIGS) {
      try {
        const result = await syncTable(db, supabase, userId, config);
        totalResult.pushed += result.pushed;
        totalResult.pulled += result.pulled;
        totalResult.errors.push(...result.errors);
      } catch (err) {
        totalResult.errors.push(`${config.localTable}: ${err instanceof Error ? err.message : String(err)}`);
      }
    }
  } finally {
    syncInProgress = false;
  }
  
  return totalResult;
}
```

`syncTable` function implements LWW:
1. Read all local rows: `db.getAllAsync(\`SELECT * FROM ${config.localTable}\`)`
2. Fetch remote rows: `supabase.from(config.remoteTable).select('*').eq('user_id', userId)`
3. Build remote lookup map keyed on `config.primaryKey`
4. **Push local-newer**: filter local rows where no remote match OR `new Date(local[config.timestampColumn]) > new Date(remote[config.timestampColumn])`. Map to include `user_id`. Upsert via `supabase.from(config.remoteTable).upsert(toUpsert, { onConflict: \`user_id,${config.primaryKey}\` })`
5. **Pull remote-newer**: for each remote row, if no local match OR remote timestamp > local timestamp, upsert into local SQLite using parameterized INSERT OR REPLACE
6. For `hasAutoIncrement: true` tables (lesson_attempts, question_attempts), use a UUID-based `remote_id` approach: generate a stable ID for remote matching instead of relying on autoincrement IDs. The local autoincrement ID stays local; remote gets a separate mapping.
7. Return `SyncResult` with counts

Important: All Supabase calls wrapped in try/catch. Network failures return error in SyncResult, never throw (per D-08 offline-first guarantee).
