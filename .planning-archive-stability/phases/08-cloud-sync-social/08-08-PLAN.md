Inside the component function body (after existing hooks, near the other useState calls), add:
```typescript
const { isAnonymous } = useAuth();
const [showAccountPrompt, setShowAccountPrompt] = useState(false);
```

Add a useEffect that triggers the prompt when entering summary stage after a passing lesson that is in ACCOUNT_PROMPT_LESSONS:
```typescript
useEffect(() => {
  if (
    stage === 'summary' &&
    quizResults?.passed &&
    isAnonymous &&
    lesson &&
    (ACCOUNT_PROMPT_LESSONS as readonly number[]).includes(lesson.id)
  ) {
    setShowAccountPrompt(true);
  }
}, [stage, quizResults?.passed, isAnonymous, lesson]);
```

Add dismiss handler that writes account_prompt_declined_at to user_profile:
```typescript
const handleAccountPromptDismiss = useCallback(async () => {
  setShowAccountPrompt(false);
  try {
    await db.runAsync(
      `UPDATE user_profile SET account_prompt_declined_at = datetime('now') WHERE id = 1`
    );
  } catch {
    // Non-critical — prompt will re-show on next qualifying lesson
  }
}, [db]);

const handleAccountPromptSignIn = useCallback(() => {
  setShowAccountPrompt(false);
  router.push('/auth');
}, []);
```

In the JSX, render AccountPrompt alongside the summary stage (after the LessonSummary component, inside the same stage === "summary" block):
```tsx
{stage === "summary" && quizResults && (
  <>
    <LessonSummary ... />
    <AccountPrompt
      visible={showAccountPrompt}
      onDismiss={handleAccountPromptDismiss}
      onSignIn={handleAccountPromptSignIn}
    />
  </>
)}
```

Note: AccountPrompt is a Modal with transparent=true, so it overlays on top of LessonSummary. It does NOT replace the summary view. The user sees the summary behind the modal backdrop.
