  <what-built>
    Phase 1 Design Foundation: polished design system components with shared animation presets, haptic feedback utility, and consistent screen transitions across all routes.
  </what-built>
  <how-to-verify>
    Start the dev server with `npm start` and open on device/emulator.

    **1. Screen Transitions (TRANS-01, TRANS-03, STATE-04):**
    - Navigate from Home tab to a lesson — should slide up from bottom (400ms, smooth)
    - Press back/complete lesson — should slide down
    - Navigate to Progress tab — should fade (300ms, no jump)
    - Navigate to onboarding (if accessible) — should fade
    - Navigate to wird-intro (if accessible) — should slide up from bottom
    - Verify: NO jarring jumps, NO content flicker, NO double-animation

    **2. Button Press (DES-02):**
    - Tap any primary button — should scale down to 0.97 with spring bounce, then back
    - The press should feel satisfying and snappy, not sluggish or flat
    - Verify: haptic tap feedback on press (light vibration)

    **3. Card Interactive (DES-02):**
    - If any cards use `interactive` prop — tap and verify gentle scale (0.98) + haptic
    - If no interactive cards visible yet — skip (downstream phases will use this)

    **4. HearButton (DES-02):**
    - Tap any audio/hear button — should have press animation (scale down + spring back)
    - Verify: haptic tap feedback on press

    **5. QuizOption (DES-02, DES-04):**
    - Enter a lesson and answer a question
    - Tap an option — should scale down with spring bounce + haptic tap
    - Get correct answer — should pulse slightly + success haptic (different from tap)
    - Get wrong answer — should shake horizontally + error haptic (different from success)
    - Verify: correct and wrong haptics feel distinct from each other and from tap

    **6. Tab Bar (DES-04):**
    - Tap between Home and Progress tabs — should have light haptic tap

    **7. Overall Feel (DES-01):**
    - Does the app feel consistent? Same animation "personality" everywhere?
    - Do transitions feel "elegant base with occasional energy" per the design direction?
  </how-to-verify>
