  <files></files>
  <read_first>

  <action>
    **Pre-build validation checks:**

    1. Run `npm run validate` — must exit 0 (no lint or typecheck errors)
    2. Run `npm test` — all tests must pass
    3. Verify `app.config.ts` has `version: "1.0.0"` and correct bundle IDs:
       - iOS: `com.tilaapp.tila`
       - Android: `com.tila.app`

    **Trigger the build:**

    Per research, use the `preview` profile first for direct device testing (production profile creates App Store-signed builds that can only install via TestFlight):

    4. Run `eas device:create` — generates a registration URL. Print the URL for the user to open on their iPhone to register the device UDID.

    5. After device is registered (or if already registered), run:
       ```bash
       eas build --platform ios --profile preview
       ```
       This creates an internally-distributed build installable on registered devices.

    6. When build completes, print the install URL / QR code that EAS provides.

    7. Also trigger the production build for App Store submission preparation:
       ```bash
       eas build --platform ios --profile production
       ```
       This creates the App Store-signed IPA. It can be uploaded to TestFlight later via `eas submit`.

    **Note:** EAS builds run in the cloud. Build time is typically 10-20 minutes. The executor should trigger the build and note the build URL for the user to monitor.
