**src/db/provider.tsx — Full rewrite per D-01 through D-04:**

1. Add a union type for init state:
```typescript
type InitState =
  | { status: 'loading' }
  | { status: 'error'; error: Error }
  | { status: 'ready'; db: SQLiteDatabase };
```

2. Replace `useState<SQLiteDatabase | null>(null)` with `useState<InitState>({ status: 'loading' })`.

3. Add `const attemptRef = useRef(0)` for stale-attempt guarding (per D-03).

4. Create an `initDb` function wrapped in `useCallback`:
   - Increment `attemptRef.current` and capture as `thisAttempt`
   - Set state to `{ status: 'loading' }`
   - Start a 15-second timeout (per D-02) that checks `attemptRef.current === thisAttempt` before setting error state
   - Call `getDatabase().then(...)` — in the `.then()`, check `attemptRef.current === thisAttempt` before proceeding, clear timeout, set `{ status: 'ready', db: database }`, call `onReady?.()`
   - Add `.catch(err => ...)` — check `attemptRef.current === thisAttempt`, clear timeout, set `{ status: 'error', error: err }`
   - Return cleanup function that clears the timeout

5. `useEffect(() => { return initDb(); }, [initDb]);`

6. Render logic:
   - `if (state.status === 'loading') return <>{fallback}</>;`
   - `if (state.status === 'error') return <ErrorFallback onRetry={initDb} />;` (per D-04)
   - `return <DatabaseContext.Provider value={state.db}>{children}</DatabaseContext.Provider>;`

7. Import `ErrorFallback` from `../components/feedback/ErrorFallback`.

8. Keep `useDatabase()` hook unchanged — it still reads from DatabaseContext and throws if null.
