<objective>
Add selective screen-level error boundaries to the lesson and home screens so a render crash in one screen shows recovery UI instead of taking down the entire app.

Purpose: The root Sentry.ErrorBoundary catches everything, but it replaces the ENTIRE app with error UI. Screen-level boundaries contain the blast radius — a lesson crash lets the user go home and continue using the app.

Output: ScreenErrorFallback component, ErrorBoundary wrappers on lesson and home screens, Sentry reporting wired, regression tests.
</objective>

This component receives `FallbackProps` from react-error-boundary (per pitfall 4 in research) and adds a "Go Home" button alongside "Try Again".

```typescript
import { View, Text, Pressable, StyleSheet } from "react-native";
import { router } from "expo-router";
import { useColors } from "../../design/theme";
import { typography, spacing, radii } from "../../design/tokens";
import type { FallbackProps } from "react-error-boundary";

export function ScreenErrorFallback({ error, resetErrorBoundary }: FallbackProps) {
  const colors = useColors();

  return (
    <View style={[styles.container, { backgroundColor: colors.bg }]}>
      <Text style={[typography.heading2, { color: colors.text }]}>
        Something went wrong
      </Text>
      <Text
        style={[
          typography.body,
          styles.message,
          { color: colors.textSoft, marginTop: spacing.md, marginBottom: spacing.xxl },
        ]}
      >
        Don't worry -- your progress is saved.
      </Text>
      <Pressable
        onPress={resetErrorBoundary}
        style={[
          styles.button,
          {
            backgroundColor: colors.primary,
            paddingHorizontal: spacing.xxl,
            paddingVertical: spacing.lg,
            borderRadius: radii.md,
            marginBottom: spacing.md,
          },
        ]}
      >
        <Text style={[typography.bodyLarge, { color: colors.white }]}>
          Try Again
        </Text>
      </Pressable>
      <Pressable
        onPress={() => router.replace("/")}
        style={[
          styles.button,
          {
            backgroundColor: "transparent",
            paddingHorizontal: spacing.xxl,
            paddingVertical: spacing.lg,
            borderRadius: radii.md,
            borderWidth: 1,
            borderColor: colors.primary,
          },
        ]}
      >
        <Text style={[typography.bodyLarge, { color: colors.primary }]}>
          Go Home
        </Text>
      </Pressable>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: "center",
    alignItems: "center",
    padding: 24,
  },
  message: {
    textAlign: "center",
  },
  button: {
    alignItems: "center",
    width: "100%",
    maxWidth: 280,
  },
});
```

2. Wrap the screen's returned JSX in an ErrorBoundary. The boundary goes INSIDE the default export function's return, wrapping all the content but INSIDE SafeAreaView (so safe area insets are preserved in the fallback). The exact placement:
```typescript
export default function LessonScreen() {
  // ... all hooks and state ...

  return (
    <SafeAreaView style={[styles.container, { backgroundColor: colors.bg }]}>
      <ErrorBoundary
        onError={(error, info) => {
          Sentry.captureException(error, {
            extra: { componentStack: info.componentStack },
          });
        }}
        FallbackComponent={ScreenErrorFallback}
      >
        {/* ... existing screen content ... */}
      </ErrorBoundary>
    </SafeAreaView>
  );
}
```

CRITICAL per D-06: The `onError` callback MUST call `Sentry.captureException` explicitly because child boundaries consume errors before the root Sentry.ErrorBoundary sees them.
