  <name>Task 1: Fix dark mode for AtmosphereBackground + font readability (Issues #1, #3)</name>
  <files>src/design/atmosphere/AtmosphereBackground.tsx</files>
  <action>
The root cause of both the light-mode onboarding (#1) and unreadable font colors (#3) is that `AtmosphereBackground.tsx` hardcodes light-mode hex colors (`#F8F6F0`, `#F2EADE`) in its `PRESETS` constant. When the app is in dark mode, the atmosphere still renders light backgrounds, making all onboarding text (which correctly uses `colors.brown`, `colors.textMuted` etc. from dark tokens) invisible against the wrong background.

Fix: Make `PRESETS` theme-aware by converting them from a static constant to a function that accepts the current `mode` ("light" | "dark").

1. Import `useTheme` from `../theme` (already imports `useColors`).

2. Create a `DARK_PRESETS` record mirroring `PRESETS` but with dark-appropriate colors:
   - `home`: linearColors `["#0F1A14", "#142019"]` (bg -> bgWarm)
   - `quiz`: linearColors `["#0F1A14", "#142019"]`
   - `sacred`: linearColors `["#142019", "#0F1A14"]` (bgWarm -> bg, reversed like light)
   - `celebration`: linearColors `["#0F1A14", "#142019"]`
   - `loading`: linearColors `["#0F1A14", "#0F1A14"]`
   - `onboarding`: linearColors `["#142019", "#0F1A14"]` (bgWarm -> bg, reversed like light)
   - All dark presets: `glowColor: "rgba(196, 164, 100, 0.2)"` (slightly dimmer gold)
   - All dark presets: reduce `glowOpacity` by ~30% compared to light (e.g., home: 0.04 instead of 0.06)
   - Keep `floatingLetters` and `floatingLetterSpeed` the same as light presets.

3. In the `AtmosphereBackground` component function:
   - Get `mode` from `useTheme()` (instead of just `useColors()`).
   - Select the correct presets: `const presets = mode === "dark" ? DARK_PRESETS : PRESETS;`
   - Use `presets[preset]` instead of `PRESETS[preset]`.

Keep `PRESETS` exported (existing API unchanged). Export `DARK_PRESETS` as well.
