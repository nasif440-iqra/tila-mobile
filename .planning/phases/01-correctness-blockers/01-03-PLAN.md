  <action>
Rewrite `recordPractice` in `useHabit` per D-08, D-09, D-10.

**Strategy:** Use `db.withExclusiveTransactionAsync()` (per research recommendation) for atomic read-modify-write. Read fresh state from DB inside the transaction instead of from the React closure. This eliminates the stale-closure race condition entirely.

Replace the current `recordPractice` function (lines 20-57) with:

```typescript
const recordPractice = useCallback(async () => {
  await db.withExclusiveTransactionAsync(async (txn) => {
    // Read fresh from DB inside transaction — not from stale React closure
    const row = await txn.getFirstAsync<{
      last_practice_date: string | null;
      current_wird: number;
      longest_wird: number;
      today_lesson_count: number;
    }>('SELECT last_practice_date, current_wird, longest_wird, today_lesson_count FROM habit WHERE id = 1');

    if (!row) return;

    const today = getTodayDateString();
    const gap = row.last_practice_date ? getDayDifference(today, row.last_practice_date) : -1;

    let newWird = row.current_wird;
    let newLongest = row.longest_wird;
    let newTodayCount = row.today_lesson_count;

    if (gap === 0) {
      // Same day — increment today's count only
      newTodayCount += 1;
    } else if (gap === 1) {
      // Consecutive day — extend streak
      newWird += 1;
      newTodayCount = 1;
    } else {
      // Gap > 1 or first practice — reset streak
      newWird = 1;
      newTodayCount = 1;
    }

    if (newWird > newLongest) {
      newLongest = newWird;
    }

    const updated: HabitState = {
      lastPracticeDate: today,
      currentWird: newWird,
      longestWird: newLongest,
      todayLessonCount: newTodayCount,
    };

    await txn.runAsync(
      `UPDATE habit SET last_practice_date = ?, current_wird = ?, longest_wird = ?, today_lesson_count = ?, updated_at = datetime('now') WHERE id = 1`,
      updated.lastPracticeDate,
      updated.currentWird,
      updated.longestWird,
      updated.todayLessonCount
    );

    // Sync React state after transaction commits
    setHabit(updated);
  });
}, [db]); // Note: dependency array is now [db] only — no longer depends on `habit`
```

**Key changes from current code:**
1. `useCallback` dependency array changes from `[db, habit]` to `[db]` — no longer reads from closure
2. Reads fresh from DB via `txn.getFirstAsync()` inside the exclusive transaction
3. Uses `txn.runAsync()` for the write (not the outer `db`) — critical for transaction isolation
4. `saveHabit` import from `../engine/progress` can be removed if no longer used elsewhere in this file (check — it is only used in `recordPractice`). Remove the `saveHabit` import and import only `type HabitState` from `../engine/progress`.

  describe('habit update computation', () => {
    // Pure logic test of the update algorithm
    function computeHabitUpdate(
      current: { currentWird: number; longestWird: number; todayLessonCount: number; lastPracticeDate: string | null },
      today: string,
      getDayDiff: (a: string, b: string) => number
    ) {
      const gap = current.lastPracticeDate ? getDayDiff(today, current.lastPracticeDate) : -1;
      let newWird = current.currentWird;
      let newLongest = current.longestWird;
      let newTodayCount = current.todayLessonCount;

      if (gap === 0) {
