    3. After prebuild completes, verify three things:

       a. Check `ios/tila/Info.plist` for `UIDeviceFamily`:
          - Run: `grep -A 5 "UIDeviceFamily" ios/tila/Info.plist`
          - Expected: array containing only `<integer>1</integer>` (iPhone)
          - FAIL if: array contains `<integer>2</integer>` (iPad)

       b. Check for absence of iPad orientation key:
          - Run: `grep "UISupportedInterfaceOrientations~ipad" ios/tila/Info.plist`
          - Expected: no match (empty output)
          - FAIL if: key exists

       c. Check Xcode project for TARGETED_DEVICE_FAMILY:
          - Run: `grep "TARGETED_DEVICE_FAMILY" ios/tila.xcodeproj/project.pbxproj`
          - Expected: `TARGETED_DEVICE_FAMILY = 1;` (iPhone only)
          - FAIL if: value is `"1,2"` (iPhone + iPad)

    4. **If any check fails** (known Expo bug #32344): Create a config plugin at `plugins/force-iphone-only.js`:
       ```javascript
       const { withInfoPlist, withXcodeProject } = require('@expo/config-plugins');

       module.exports = function forceIphoneOnly(config) {
         config = withInfoPlist(config, (config) => {
           config.modResults.UIDeviceFamily = [1];
           delete config.modResults['UISupportedInterfaceOrientations~ipad'];
           return config;
         });
         return config;
       };
       ```
       Then add `"./plugins/force-iphone-only"` to the `plugins` array in `app.config.ts` and re-run prebuild to verify.

    5. Clean up generated native directories after verification:
       - Run: `rm -rf ios/ android/` (these are generated files, not committed)
