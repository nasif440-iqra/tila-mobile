## Spacing Scale

Base system defined in `src/design/tokens.ts` (8px base rhythm). Phase 4 introduces no new spacing tokens.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon gaps, inline padding, label margins |
| sm | 8px | Compact element spacing, feedback panel gaps |
| lg | 16px | Section padding, card inner sections, exercise content gaps |
| xl | 24px | Screen horizontal padding, section vertical gaps, letter row gaps |
| xxl | 32px | Lesson summary vertical padding, major section breaks |
| xxxl | 48px | Screen top padding (LessonIntro) |

**Named exception -- `md: 12px`:** Pre-existing token in `src/design/tokens.ts` that does not fall on the 8px grid. Phase 4 usage is limited to inherited component internals: progress bar counter gap (QuizProgress), options grid gap (QuizQuestion, ComprehensionExercise), stage badge horizontal padding (LessonHybrid), exercise bottom-actions gap (BuildUpReader), and other pre-existing patterns from the current codebase. All new layout spacing in Phase 4 uses grid-aligned tokens.

Non-token size exceptions:
- Letter circles: 120px (normal), 88px (small, 3+ letters) -- display-size elements
- Icon circle (LessonSummary): 80px diameter -- result icon container
- Options grid max-width: 340px -- prevents options from stretching on wide screens
- Exercise max-width: 430px (GuidedReveal), 340px (BuildUpReader, FreeReader) -- content readability constraint
- Progress bar track height: 6px (QuizProgress), 8px (LessonHybrid) -- fine-grained sizing below token scale
- GuidedReveal chips: 76px min-width/min-height -- touch target for form display

## Visual Specifications

### Lesson Intro Screen

**Layout:** Full screen, vertically centered content with header at top and CTA at bottom.

**Header region:**
- Phase label: `caption` preset, uppercase, letterSpacing 1, color `textMuted`, centered
- Content: "Phase {N} . Module {M}" (existing format)
- Title: `heading1` preset, color `text`, centered
- Entrance: FadeIn duration `slow` (400ms)

**Letter cards:**
- Circle: 120px diameter (88px when 3+ letters), borderRadius 50%, background `primarySoft`
- Behind each circle: WarmGlow (animated, size: 160px for normal / 120px for small, color: `accentGlow`, pulseMin: 0.05, pulseMax: 0.15) -- sets warm, inviting tone
- Arabic letter: `arabicDisplay` (or `arabicLarge` for small), color `primaryDark`
- Letter name below: `bodySmall`, color `textSoft`, marginTop `xs`
- HearButton below name: marginTop `sm`, size 36px
- Row layout: flexDirection row, gap `xl`, centered
- Entrance: FadeIn delay 150ms, duration 500ms, with staggered scale from 0.9 to 1.0 using `springs.gentle` (each card delayed by `staggers.fast.delay` = 50ms)

**Instruction text:**
- `body` preset, color `textMuted`, centered
- Content: "Tap to hear the sound, then start the quiz" (sound mode) / "Tap to hear, then start the quiz" (name mode)

**Family rule (optional):**
- `bodySmall` preset, color `textSoft`, centered, maxWidth 300
- lineHeight 20

**CTA button:**
- Full width, `Button` primary variant
- Label: "Start Quiz"
- Entrance: FadeIn delay 300ms, duration 400ms

### Quiz Screen

**Progress bar (QuizProgress):**
- Track: full width, height 6px, borderRadius 3px, background `border`
- Fill: animated width via `springs.gentle`, background `primary`
- Near completion (>85%): fill color transitions to `accent` over `durations.normal` (300ms) -- visual signal that finish is near
- Counter: `caption` preset, fontWeight 700, color `textSoft`, right-aligned, minWidth 40px
- Layout: Row with gap `md`, marginBottom `lg`

**Streak banner:**
- Trigger: Streak milestones (3, 5, 7)
- Container: position absolute, top `xxxl`, left/right `xl`, background `accentLight`, border 2px `accent`, borderRadius `lg`
- Text: `bodyLarge` preset, fontWeight 700, color `accent`
- Entrance: SlideInDown with `springs.bouncy` (stiffness 300, damping 20)
- Exit: SlideOutUp duration 300ms
- Duration: 1500ms, then auto-dismiss
- Copy: "{N} in a row! {message}" where message is from getStreakMessage pool
- Haptic: `hapticSuccess` when banner appears

**Recycled question hint:**
- `caption` preset, italic, centered, color `textMuted`
- Copy: "Review -- missed questions come back once"
- marginTop -`sm`, marginBottom `sm`

### Quiz Question

**Prompt area (centered):**
- Audio questions: HearButton size 72px + prompt `bodyLarge` fontWeight 700 color `brown` + "Replay" link (`bodySmall` fontWeight 600 color `primary`)
- Letter-to-sound: `arabicDisplay` letter + optional subtext `body` fontWeight 600 color `textSoft` + HearButton 44px
- Letter-to-name: `arabicDisplay` letter + optional subtext
- Visual questions: prompt `bodyLarge` fontWeight 700 color `brown`
- marginBottom `xl` below prompt area

**Options grid:**
- 2x2 grid: flexDirection row, flexWrap wrap, gap `md`, maxWidth 340px
- Each option: 47% width
- Uses `QuizOption` component (Phase 1 polished -- springs, haptics, correct/wrong animations)
- No changes to QuizOption itself

**Correct feedback pill:**
- Container: row, paddingVertical `sm`, paddingHorizontal `lg`, borderRadius full, border `normal` (1.5px) `primary`, background `primarySoft`, marginTop `lg`
- Text: `bodySmall` fontWeight 700, color `primary`
- Content: checkmark + " Correct!"
- Entrance: FadeIn duration 300ms
- Enhancement: brief scale pulse from 0.95 to 1.0 using `springs.press` on appear
- Haptic: handled by QuizOption (leaf component owns haptic -- no duplicate)

### Wrong Answer Panel

**Container:**
- SlideInDown entrance with `springs.bouncy` (stiffness 300, damping 25)
- Background: `dangerLight`, borderRadius `xl`, padding `lg`, gap `md`
- Shadow: `shadows.card`

**Explanation row:**
- X icon: 18px fontWeight 700, color `danger`
- Text: `bodySmall` fontWeight 600, lineHeight 20, color `dangerDark`
- Fallback copy: "The correct answer is {name} ({letter})" (existing)
- Enhancement: Randomly select from WRONG_ENCOURAGEMENT pool as prefix: e.g. "Almost -- the details make the difference. The correct answer is..."

**Letter comparison (visual questions):**
- Row: chosen letter (arabicLarge, color `danger`) arrow (body, color `textMuted`) correct letter (arabicLarge, color `primary`)
- Names below each: `caption` fontWeight 700

**Continue button:**
- `Button` primary variant, label "Got It"
- Haptic: `hapticTap` on press (handled by Button component)

**Tone:** The panel should feel supportive, not punishing. The `dangerLight` (#FCE6E5) background is warm pink, not aggressive red. The copy is encouraging and educational.

### Mid-Quiz Celebration (QuizCelebration)

**Overlay:**
- Full screen absolute fill, background `bg` at 90% opacity
- zIndex 200
- Entrance: FadeIn duration 300ms
- Exit: FadeOut duration 300ms

**Content (centered):**
- Star emoji: 56px, marginBottom `lg`
- Title: `heading1` preset, color `primary`
- Copy: Randomly selected from MID_CELEBRATE_COPY.default (or .harakat for harakat lessons)
- Subtitle: `body` preset, color `textSoft`, marginBottom `xl`
- Copy: "You're halfway there" (existing) -- or context-specific from engagement.js
- Tap instruction: `caption` preset, color `textMuted`, "Tap to continue"
- Enhancement: Scale entrance from 0.9 to 1.0 using `springs.bouncy` on the content container
- Haptic: `hapticMilestone` on appear

### Lesson Summary Screen

**Layout:** Full screen centered, maxWidth 380px card with shadows.card, borderRadius `xl`, padding `xxl` vertical / `lg` horizontal.

**Result icon:**
- Circle: 80px diameter, borderRadius 40, background `primarySoft` (passed) / `accentLight` (retry)
- Content: checkmark or refresh unicode glyph, 36px, color `primary` (passed) / `accent` (retry)
- marginBottom `xl`

**Accuracy display:**
- Animated count-up: 0 to {percentage}% over 800ms with `easings.contentReveal` (Easing.out(cubic))
- Font: 56px, lineHeight 64, `headingBold`, fontWeight 700
- Color: Transitions based on final percentage:
  - >=80%: `primary` (#163323)
  - >=50%: `accent` (#C4A464)
  - <50%: `danger` (#BD524D)
- marginBottom `lg`
- Entrance: FadeIn delay 200ms, duration 400ms

**Performance message:**
- Headline: `heading2` preset, color `text`, centered
- Copy: From COMPLETION_HEADLINES[tier] (existing engagement.js)
- Subline: `body` preset, color `textSoft`, centered, maxWidth 300
- Copy: From COMPLETION_SUBLINES[tier] (existing engagement.js)
- marginBottom `lg`
- Entrance: FadeIn delay 400ms, duration 400ms

**Score:**
- `bodySmall` preset, color `textMuted`, centered
- Copy: "{correct} out of {total} correct"
- marginBottom `xl`

**Closing quote:**
- Font: 14px `headingItalic`, lineHeight 22, italic, color `textMuted`, centered
- Content: Random from CLOSING_QUOTES (existing engagement.js)
- marginBottom `xxl`

**Celebration -- score-proportional visual warmth:**
- Strong (>=80%): WarmGlow behind icon circle (size: 140px, animated, pulseMin: 0.08, pulseMax: 0.22, color: `accentGlow`), hapticMilestone on mount
- Partial (50-79%): WarmGlow at lower intensity (size: 100px, pulseMin: 0.04, pulseMax: 0.10), hapticSuccess on mount
- Weak (<50%): No WarmGlow, hapticTap on mount (gentle acknowledgement, not punishment)

**Action buttons:**
- Passed: "Continue" -- `Button` primary
- Failed: "Try Again" -- `Button` primary, followed by "Continue Anyway" -- `Button` ghost, separated by `sm` (8px) spacer
- Entrance: FadeIn delay 900ms, duration 400ms

### Exercise Screens -- Shared Patterns

All 6 exercise types share these visual conventions:

**Container:**
- flex: 1, centered content, paddingHorizontal `lg` or `xl`
- maxWidth constraint (340-430px) for readability

**Instruction text:**
- `body` preset, color `textSoft`, centered, at top or above interactive area

**Primary Arabic display:**
- `arabicDisplay` for primary focus element
- `arabicLarge` for secondary Arabic (form variants, comparison)
- `arabicBody` for interactive letter boxes

**Feedback panels:**
- Correct: background `primarySoft`, text color `primary`
- Wrong: background `dangerLight`, text color `dangerDark` / `danger`
- Entrance: FadeInDown duration 300ms

**CTA buttons:**
- Always at bottom of exercise area
- "Continue" or "Next" -- `Button` primary variant
- Contextual labels: "Next Form ->" (GuidedReveal), "I can read it ->" (BuildUpReader), "Got it" (wrong answer), checkmark + "I read it right" (FreeReader)

**Haptic pattern (all exercises):**
- Replace all direct `Haptics.impactAsync(Light)` with `hapticTap()`
- Replace all direct `Haptics.notificationAsync(Success)` with `hapticSuccess()`
- Replace all direct `Haptics.notificationAsync(Error)` with `hapticError()`
- No new haptic patterns -- use existing presets exclusively

### Exercise -- GuidedReveal Specifics

**Letter header:** Arabic letter `arabicDisplay` + HearButton, centered, gap `sm`
- Enhancement: WarmGlow behind letter (size: 120px, color: `accentGlow`, pulseMin: 0.04, pulseMax: 0.12) for visual warmth

**Position chips:**
- Row layout, gap `md`, centered
- Each chip: minWidth 76px, minHeight 76px, borderRadius `lg`, border 2px
- Current: background `primary`, text white, scale 1.05
- Revealed: background `bgCard`, border `primary`, shadow `card`
- Unrevealed: transparent, border dashed `border`, opacity 0.45, scale 0.95
- Staggered entrance: FadeIn with 100ms delay per chip

**Explanation card:** border 1px `primary`, background `primarySoft`, borderRadius `lg`, padding `lg`

**Context word card:** border 1px `border`, background `bgCard`, borderRadius `xl`, padding `xl`, shadow `card`, centered

### Exercise -- TapInOrder Specifics

**Letter boxes:**
- 60x60px, borderRadius `md`, border 2px
- Default: background `bgCard`, border `border`
- Next (hint): border `accent`, accentGlow shadow (opacity 0.5, radius 8, elevation 4)
- Tapped: background `primary`, border `primary`, text white
- Wrong tap: Shake animation using existing withSequence pattern (4px, 50ms steps)

**Layout:** flexDirection row-reverse (RTL reading order), gap `md`

**Completion banner:** FadeInDown 300ms, background `primarySoft`, borderRadius `md`, text `primary` fontWeight 600

### Exercise -- BuildUpReader Specifics

**Step dots:** Row of 6px circles, gap 6px, filled `primary` (completed) / `border` (upcoming)

**Arabic display:** Key step transition with FadeInDown springify per step

**Sound labels:** Row-reverse (RTL), current segment highlighted with `primarySoft` background

**Context card:** background `accentLight`, border 1px `border`, borderRadius `md`, centered content

### Exercise -- FreeReader Specifics

**Self-assessment flow (3 states):**
1. Initial: "Hear the correct reading" -- primary button
2. Heard: checkmark + "I read it right" (secondary) + "Hear it again" (ghost)
3. Retry: "Listen carefully and try again" (caption) + "Hear again" (ghost) + checkmark + "Got it now" (secondary)

**Context badge:** Pill shape (borderRadius full), background `accentLight`, border 1px `border`, horizontal content with transliteration + meaning + surah ref

### Exercise -- SpotTheBreak Specifics

**Segment buttons:** Row-reverse (RTL), each with borderRadius `md`, border 2px, paddingHorizontal `lg`, paddingVertical `sm`
- Correct tap: border `primary`, background `primarySoft`
- Wrong tap: border `danger`, background `dangerLight` + shake animation

**Feedback panel:** FadeInDown 300ms, borderRadius `md`, padding `lg`, gap `md`

### Exercise -- ComprehensionExercise Specifics

**Options layout:** Vertical stack (not 2x2 grid), full width, gap `md`
- Uses QuizOption component with staggered FadeInDown (70ms between options, springify)

**Wrong feedback panel:** background `dangerLight`, borderRadius `lg`, padding `lg`, gap `md`
