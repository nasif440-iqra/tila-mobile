## BismillahOverlay (Lesson Entry)

**Context:** Shown before the first lesson of each app session (D-08). Session detection via `expo-secure-store` — store timestamp on each Bismillah display, compare against app foreground timestamp.

**Behavior:**
1. When `app/lesson/[id].tsx` mounts, check if Bismillah has been shown this session
2. If not shown: render BismillahOverlay as absolute-positioned layer over lesson content
3. Overlay displays Bismillah with same visual treatment as BismillahMoment step
4. Auto-fades out after 2.5 seconds (FadeOut duration 500ms)
5. Mark session as having shown Bismillah in SecureStore
6. No skip button (D-10)

**Visual:** Identical to BismillahMoment step but rendered as overlay with `bgWarm` background at opacity 0.97 (slight transparency to hint at content beneath).

## Branded Asset Integration

### Logo Rendering Strategy

**Decision:** Convert SVG logo files to React Native components using `react-native-svg`.

| Asset | File | Rendering | Usage |
|-------|------|-----------|-------|
| Transparent mark | `assets/logo/tila-transparent-mark.svg` | Extract path/circle elements into a React component using `Svg`, `Path`, `Circle` from `react-native-svg` | Welcome screen — replaces inline `LogoMark` component |

**Why not use `react-native-svg-transformer`?** The branded SVGs contain CSS `@keyframes` animations that are not supported in React Native SVG rendering. The animations (glow pulse, moon breathe, star sparkle) must be reimplemented using Reanimated.

**Logo component approach:**
1. Create `src/components/onboarding/BrandedLogo.tsx`
2. Extract static SVG shapes (crescent, stars, arch, keystone) from `tila-transparent-mark.svg`
3. Reimplement animations using Reanimated: `withRepeat(withSequence(withTiming))` for glow pulse, star sparkle
4. Use `colors.primary` and `colors.accent` tokens (not hardcoded hex values) so theming works
5. Component accepts `width` and `height` props, scales internally via `viewBox`

### Animations to Reimplement from SVG

| SVG Animation | Reanimated Equivalent | Duration | Behavior |
|---------------|----------------------|----------|----------|
| `glowPulse` (opacity 0.04-0.12) | `withRepeat(withSequence(withTiming(0.12), withTiming(0.04)))` | 4s cycle | Continuous, infinite |
| `moonBreathe` (translateY 0 to -2px, brightness) | `withRepeat(withSequence(withTiming(-2), withTiming(0)))` | 5s cycle | Continuous, skip brightness (not supported in RN) |
| `sparkle` (opacity 0.2-1.0, scale 0.7-1.4) | `withRepeat(withSequence(withTiming({opacity, scale})))` | 3s cycle | Staggered start per star (0, 0.7s, 1.4s, 2.1s, 1.0s) |
| `archGlow` (opacity 0.5-0.9) | `withRepeat(withSequence(withTiming))` | 4s cycle | Staggered (0.3s, 0.8s) |
| `keystonePulse` (r 4-5, opacity 0.7-1.0) | `withRepeat(withSequence(withTiming))` via scale | 3s cycle | Scale 1.0-1.25 approximation |

**Performance constraint:** All logo animations run on UI thread via Reanimated. Maximum 8 simultaneous animated values for the logo (2 glows + 5 stars + 1 keystone). This is well under the 20-component cap from Phase 1 pitfalls.

## Performance Constraints

| Constraint | Limit | Rationale |
|------------|-------|-----------|
| Simultaneous animated components per screen | 20 max | Phase 1 pitfall: mid-range Android drops to 15-30fps beyond this |
| Animated properties | `transform` + `opacity` only | Never animate width, height, margin, padding |
| FloatingLettersLayer | 12 letters | Already at limit — do not add more |
| WarmGlow pulsing | Single animated opacity value per glow | Minimal overhead |
| Logo animations | 8 animated values max | 2 glows + 5 stars + 1 keystone |
| Total per step max | ~15 animated values | Logo (8) + content stagger (4-5) + WarmGlow (1-2) |
