<task type="auto">
  <name>Task 2: Add iOS privacy manifest configuration to app.config.ts</name>
  <files>app.config.ts</files>
  <read_first>app.config.ts</read_first>
  <action>
    Add `privacyManifests` to the `ios` section of app.config.ts, right after `bundleIdentifier`. This is a safety net per D-15 — Expo SDK packages bundle their own PrivacyInfo.xcprivacy but Apple sometimes fails to parse them from static CocoaPods dependencies.

    The `ios` block should become:

    ```typescript
    ios: {
      supportsTablet: false,
      bundleIdentifier: "com.tilaapp.tila",
      privacyManifests: {
        NSPrivacyAccessedAPITypes: [
          {
            NSPrivacyAccessedAPIType: "NSPrivacyAccessedAPICategoryUserDefaults",
            NSPrivacyAccessedAPITypeReasons: ["CA92.1"],
          },
        ],
      },
      infoPlist: {
        UIBackgroundModes: [],
        ITSAppUsesNonExemptEncryption: false,
      },
    },
    ```

    UserDefaults is declared because React Native and most RN libraries use NSUserDefaults. Reason code CA92.1 covers "accessing user defaults to read and write information that is only accessible to the app itself."

    Do NOT modify any other section of app.config.ts.
  </action>
  <verify>
    <automated>npm run validate</automated>
  </verify>

    - npm run validate passes (lint + typecheck)
  </acceptance_criteria>
