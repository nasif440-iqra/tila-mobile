<task type="auto">
  <name>Task 1: Configure eas.json with ascAppId and trigger production builds</name>
  <files>eas.json</files>
  <read_first>eas.json, .planning/phases/06-app-store-submission/06-02-SUMMARY.md</read_first>
  <action>
    1. Read the ascAppId from Plan 02's SUMMARY (provided by founder after creating App Store Connect entry).

    2. Update eas.json to add the ascAppId under the submit.production section:
    ```json
    "submit": {
      "production": {
        "ios": {
          "ascAppId": "THE_ID_FROM_FOUNDER"
        }
      }
    }
    ```

    3. Trigger production builds for both platforms:
    ```bash
    eas build --platform ios --profile production --non-interactive
    eas build --platform android --profile production --non-interactive
    ```

    These build on EAS cloud servers (not locally). Build takes ~15-30 minutes per platform. The `--non-interactive` flag avoids prompts. The `autoIncrement: true` in eas.json handles version bumping.

    4. Wait for builds to complete. Check status with:
    ```bash
    eas build:list --limit 2
    ```

    5. Once both builds succeed, note the build URLs/IDs for the submission tasks.
  </action>
  <verify>
    <automated>eas build:list --limit 2 --json 2>/dev/null | head -20</automated>
  </verify>
  <acceptance_criteria>
    - grep -q "ascAppId" eas.json returns 0
    - eas build:list shows two recent production builds (ios + android) with status "finished"
  </acceptance_criteria>
  <done>
    - eas.json updated with ascAppId
    - Production iOS build completed on EAS
    - Production Android build completed on EAS
  </done>
</task>
