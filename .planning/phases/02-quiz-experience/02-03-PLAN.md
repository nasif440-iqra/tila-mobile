## Shared-Token Blast Radius

This plan modifies shared design tokens that are consumed beyond the quiz screen. Executor must be aware:

**Line-height corrections (tokens.ts) are GLOBAL typography changes:**
- `arabicDisplay` (lineHeight 100 -> 158): Used by ArabicText size="display" anywhere in the app. Screens using display-size Arabic text will have more vertical space. This is a CORRECTION (the values were reverted/lost), not a new change -- Phase 1 established these ratios for diacritic clearance.
- `arabicLarge` (lineHeight 54 -> 72): Used by ArabicText size="large" in quiz options (pre-quizOption), lesson cards, and other surfaces.
- `arabicBody` (lineHeight 36 -> 48): Used by ArabicText size="body" in explanatory text, lesson content.

These lineHeight increases are REQUIRED for Arabic diacritics to render without clipping. They were established in Phase 1 and lost during the worktree collision. Non-quiz screens that use these tiers will gain vertical space -- this is intentional and correct.

**overflow: "visible" on ArabicText:**
- Applied globally to all ArabicText instances. Needed for tall diacritics (hamza, shadda stacking) to render outside the text bounding box.
- In constrained layouts (e.g., quiz option cards with fixed heights, comparison rows), this could cause visible overflow if the container clips. However, quiz option cards use flexible height and the lineHeight increase provides the space. The executor should visually verify that ArabicText with overflow: visible does not bleed into adjacent elements in option cards.

**drift token restoration is SHARED-SYSTEM REPAIR, not Phase 2 behavior:**
- The `drift` token is consumed by `FloatingLettersLayer.tsx` (Phase 1 atmosphere component), not by any quiz component.
- It is restored here because it was lost in the same worktree collision that broke the Phase 2 tokens.
- Without it, FloatingLettersLayer.tsx fails typecheck (TS2305: Module has no exported member 'drift').
- This is infrastructure repair necessary for a clean build, not Phase 2 scope expansion.

   Actually, the cleaner approach: just put the opacity into the existing `animatedStyle` worklet. Change the existing `animatedStyle` from:
