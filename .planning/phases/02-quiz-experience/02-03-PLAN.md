**QuizOption.tsx — Move wrongOpacity into useAnimatedStyle worklet:**

The problem: line 169 reads `wrongOpacity.value` directly in a style prop passed to AnimatedPressable. Reanimated shared values must be read inside `useAnimatedStyle` to animate on the UI thread.

Fix approach: Create a new `useAnimatedStyle` for the container opacity that handles all three opacity states (dimmed, wrong, normal), and apply it as an animated style instead of inline.

1. Add a new animated style hook after the existing `glowStyle` (around line 108):
   ```typescript
   const containerOpacityStyle = useAnimatedStyle(() => ({
     opacity: isDimmed ? 0.35 : isWrong ? wrongOpacity.value : 1,
   }));
   ```

   Wait — `isDimmed` and `isWrong` are derived from `state` which changes each render. The worklet captures the closure. This is fine because useAnimatedStyle re-evaluates when the component re-renders (state change triggers re-render, which re-creates the worklet with new isDimmed/isWrong values), AND it tracks wrongOpacity.value reactively for the animation within a single state.

   Actually, the cleaner approach: just put the opacity into the existing `animatedStyle` worklet. Change the existing `animatedStyle` from:
   ```typescript
   const animatedStyle = useAnimatedStyle(() => ({
     transform: [{ scale: scale.value }, { translateX: translateX.value }],
   }));
   ```
   to:
   ```typescript
   const animatedStyle = useAnimatedStyle(() => ({
     transform: [{ scale: scale.value }, { translateX: translateX.value }],
     opacity: wrongOpacity.value,
   }));
   ```

2. Then change the inline style on AnimatedPressable (line ~169) from:
   ```typescript
   opacity: isDimmed ? 0.35 : isWrong ? wrongOpacity.value : 1,
   ```
   to:
   ```typescript
   opacity: isDimmed ? 0.35 : 1,
   ```

   This way: `isDimmed` (a boolean from JS render) controls the static dimmed case. The animated `wrongOpacity` case is handled by the worklet in `animatedStyle`. When state is "default", wrongOpacity resets to 1 (line 98), so the worklet opacity is 1. When state is "selectedWrong", wrongOpacity animates 1 -> 0.5 -> 0.7, and the worklet picks it up on the UI thread.

   The `isDimmed ? 0.35 : 1` in inline style still works because for dimmed options, wrongOpacity stays at 1 (reset in the else branch), so the animatedStyle opacity is 1, and the inline 0.35 takes precedence via style cascading. For non-dimmed, non-wrong states, both are 1. For wrong state, inline is 1 but animatedStyle provides the animated value.

   Important: The `animatedStyle` must come AFTER the inline style object in the style array so that the animated opacity from the worklet overrides the inline `opacity: 1` during wrongOpacity animation. Verify the style array order is:

     styles.base,

   ]}
