export interface PhraseWord {

**RevealWord internal component** (not exported -- each instance owns its own shared values to comply with React hook rules):
- Props: `word: PhraseWord`, `index: number`, `staggerDelay: number`, `wordDuration: number`, `skipImmediate: boolean`
- Uses `useSharedValue` for `arabicOpacity`, `translitOpacity`, `meaningOpacity` (all start at `skipImmediate ? 1 : 0`)

  - If `word.meaning`: `meaningOpacity.value = withDelay(index * staggerDelay + 400, withTiming(1, { duration: wordDuration * 0.7, easing: Easing.out(Easing.cubic) }))`

**Addresses review concern -- Arabic layout stability on narrow screens:**
Each RevealWord container must set `writingDirection: 'rtl'` on the View wrapping the ArabicText to ensure RTL rendering. For the horizontal layout container, use `flexWrap: 'wrap'` with `flexDirection: 'row-reverse'` (RTL natural order) instead of `flexDirection: 'row'` so Arabic words flow right-to-left. Each word unit must be a single atomic View (alignItems: 'center') so Arabic + transliteration never get separated across lines. The word unit View should NOT have flexShrink -- use `flexShrink: 0` to prevent the system from collapsing a word unit to fit.

**Addresses review concern -- Layout space reservation to prevent reflow:**
The PhraseReveal container (both horizontal and vertical) must use a fixed `minHeight` to reserve space before words reveal. For vertical layout: `minHeight: words.length * 80` (approximate per-unit height: Arabic 36px lineHeight 72px + transliteration 22px + gaps). For horizontal layout: `minHeight: 120` (Arabic lineHeight 72px + transliteration 22px + gaps). This prevents the container from growing as words appear, avoiding layout jumps for content below (like CTA buttons). The minHeight is a floor -- content can exceed it naturally.

**Addresses review concern -- Timer cleanup on unmount:**
The PhraseReveal parent component MUST properly clean up timers. Use a `useRef<ReturnType<typeof setTimeout> | null>(null)` for the completion timer. The `useEffect` that starts the timer MUST return a cleanup function: `return () => { if (timerRef.current) clearTimeout(timerRef.current); };`. The `handleSkip` function must also clear the timer before calling onComplete. This prevents stale `onComplete` callbacks firing after the component unmounts (e.g., user navigates away mid-reveal) or double-firing after skip.
